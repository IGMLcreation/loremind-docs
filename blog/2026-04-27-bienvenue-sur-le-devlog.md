Ce devlog est l'endroit où je raconte la fabrication de **LoreMind** : les features qui arrivent, les choix de design, les coulisses du projet, et de temps en temps les détours techniques.

L'idée : un rendez-vous régulier, écrit, sans hype ni jargon — juste ce qui se passe dans le projet, et pourquoi.

{/* truncate */}

## Pourquoi un devlog ?

LoreMind est un projet solo, open source, avec une vision assez précise : un outil de jeu de rôle qui respecte la **souveraineté de tes données** (IA 100% locale via Ollama, pas de cloud), pensé **par un MJ pour les MJ**, et pas par une startup qui découvre le JDR.

Mais une vision, ça se raconte. Un changelog Git, c'est utile pour les développeurs ; ça ne donne pas envie à un MJ de tester l'outil. Le devlog, c'est le pont entre le code et la table de jeu.

## Ce que tu trouveras ici

- 🎲 **Coulisses** — pourquoi tel choix d'architecture, tel parti pris d'UX
